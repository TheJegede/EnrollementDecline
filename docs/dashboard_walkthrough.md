# Dashboard Walkthrough

Six-page Streamlit application: `streamlit run streamlit_app.py`

## Page 1 — Executive Overview (`streamlit_app.py`)

Entry point. Four KPI cards across the top:

| Card | Source | Value (as built) |
|------|--------|-----------------|
| US Enrollment Forecast 2030 | `forecasts.csv`, SARIMA | ~21.3M (delta vs last actual) |
| High-Priority Leads (score ≥ 70) | `lead_predictions.csv` | ~15,000 of 50,000 leads |
| Predicted Enrollments (prob ≥ 60%) | `yield_predictions.csv` | ~2,300 of 8,860 admits |
| Chatbot Deflection Rate | `rag_eval_raw.csv` | 77% (23/30 queries handled) |

Below the KPIs:
- **Enrollment Funnel** (left) — Plotly funnel chart: Leads → Applicants → Admits → Enrolled with percent-of-initial labels. Sub-metrics show conversion rates at each stage.
- **Demographic Enrollment Shift** (right) — horizontal bar chart of projected % change from last actual year to 2035 by race/ethnicity group (White, Black, Hispanic, Asian, TwoOrMore). Red = decline, green = growth.

Module quick-links row at the bottom navigates to each of the five detail pages.

## Page 2 — Forecasting (`pages/1_Forecasting.py`)

- Interactive Plotly line chart with SARIMA and Prophet forecasts for `total_enrollment` and `hs_graduates`
- Confidence band toggle (80% and 95% intervals)
- Segment selector for `race_enrollment` breakdown
- Export forecast CSV button

## Page 3 — Lead Scoring (`pages/2_Lead_Scoring.py`)

- Sidebar filters: institution segment, minimum score, top-N limit
- Score distribution histogram with threshold line
- Average score by segment grouped bar chart
- Sortable lead table with background-gradient score column
- Per-lead SHAP waterfall: click any row to see top-3 feature contributions
- CSV download button

## Page 4 — Yield Prediction (`pages/3_Yield_Prediction.py`)

- Yield probability distribution with outreach threshold slider
- Predicted vs actual yield rate by institution segment (grouped bar)
- Predicted class composition (pie charts: by institution segment and race/ethnicity)
- Per-student SHAP detail panel
- Class size forecast with count at current threshold

## Page 5 — Chatbot (`pages/4_Chatbot.py`)

- Groq + ChromaDB streaming chat, message history preserved in `st.session_state`
- 6 sample question chips (one click populates input)
- Source citations in collapsible expander below each response
- Sidebar: clear conversation button, average latency metric
- Falls back to FAISS index if ChromaDB unavailable (Streamlit Cloud)

## Page 6 — Model Performance & Bias Audit (`pages/5_Model_Performance.py`)

Four tabs:

**Model Metrics** — test-set performance table + ROC-AUC bar chart for both ML models (XGBoost and Logistic Regression variants).

**Bias Audit** — demographic parity and disparate impact ratio tables for Lead and Yield models across `race_ethnicity`, `gender`, `first_gen_flag`, `income_band`. Flagged = DI outside 0.80–1.25 or parity gap > 5pp. Mitigation note at bottom.

**Calibration** — Platt scaling calibration curve images (`lead_calibration.png`, `yield_calibration.png`) generated by notebooks 03 and 04.

**Chatbot Eval** — four KPIs (trick refusal rate, deflection rate, mean latency, mean cosine score), latency and retrieval score tables by category, per-question latency scatter plot, raw data expander.

## Deployment

```bash
# Local
streamlit run streamlit_app.py

# Streamlit Cloud
# 1. Push repo to GitHub
# 2. Connect at share.streamlit.io
# 3. Add GROQ_API_KEY in app secrets
# 4. ChromaDB not persisted on Cloud — FAISS fallback activates automatically
```

ChromaDB note: Streamlit Cloud does not persist files between deployments. The FAISS pickle (`data/vector_db/faiss.index` + `faiss_meta.pkl`) should be committed to the repo as fallback — `src/rag/retrieval.py` detects ChromaDB failure and falls back automatically.
